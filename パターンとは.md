# パターン・ランゲージとは

## 1. 概要

パターン・ランゲージとは、ある領域における「よい実践」や「成功事例」の背後にある**本質的なコツ**を抽出し、再利用可能な形式で記述した**知の体系**です。

元々は建築家クリストファー・アレグザンダーが、よい建築・まちづくりのパターンを言語化したことに始まります。現在では、ビジネス、教育、コミュニティづくりなど、様々な領域で活用されています。

---

## 2. パターンの基本構造（共通フォーマット）

すべてのパターンは、以下の要素で構成されます。

### 必須要素

| 要素名 | 英語名 | 説明 |
|--------|--------|------|
| **番号** | `Number` | パターンの識別番号（例: No.10, 01）|
| **パターン名** | `Name` | その知恵を呼び表すタイトル |
| **導入文 / キャッチフレーズ** | `Intro` | パターンの本質を一言で表す短い文 |
| **イラスト** | `Illustration` | 概念を視覚的に表現した一枚絵 |
| **状況** | `Context` | このパターンが適用される場面・背景 |
| **問題（しかし）** | `Problem` | その状況で起こりがちな困難・障壁 |
| **解決（そこで）** | `Solution` | 問題を解決するための行動・コツ（核心） |
| **結果** | `Consequence` | 実践した後に期待される未来像 |

### 任意要素

| 要素名 | 英語名 | 説明 |
|--------|--------|------|
| **エピソード / ノート** | `Note` | 具体的な実践例や背景情報 |
| **関連パターン** | `RelatedPatterns` | 他のパターンとの関係性 |
| **タグ / カテゴリ** | `Tags` | 分類のためのラベル |

---

## 3. 記述フォーマット

パターン内のテキストは、以下のマーカーで構造化されます：

```
▼ その状況において
（状況の説明）

しかし〜 / ▼問題
（問題の説明）

▼ そこで
（解決策の説明）

▼ その結果
（期待される結果）
```

---

## 4. 表現形態の比較

あなたが作成した2つのパターン・ランゲージは、異なる表現形態を持っています。

### A. 楽天主義実践パターン（本形式）

| 特徴 | 詳細 |
|------|------|
| **形態** | 見開き2ページの本 |
| **左ページ** | パターン名、導入文、イラスト、ノート欄 |
| **右ページ** | 状況→問題→解決→結果の詳細テキスト |
| **情報量** | 豊富（詳細な説明が可能） |
| **読み方** | 順番に読み進める「物語的」体験 |
| **適用場面** | じっくり学ぶワークショップ、研修 |

### B. 燕市まちづくりパターン（カード形式）

| 特徴 | 詳細 |
|------|------|
| **形態** | 1枚のカード |
| **上部** | 番号、パターン名（大きく）、イラスト |
| **下部** | 状況→解決のエッセンス |
| **情報量** | 凝縮（要点のみ） |
| **読み方** | 手に取って眺める「カジュアル」な体験 |
| **適用場面** | 対話ツール、アイスブレイク、日常的な参照 |

### 共通点

- **イラストが視覚的アンカー**として中心的な役割を担う
- **パターン名**が大きく目立つ配置
- 「状況 → そこで」という問題解決の流れ

## 5. データモデル（型定義の基礎）

```typescript
// パターン・ランゲージ全体（1つの冊子・カードセット）
interface PatternLanguage {
  id: string;                    // 例: "rakuten", "tsubame"
  name: string;                  // 例: "楽天主義実践パターン"
  description: string;           // ランゲージ全体の説明
  themeColor: string;            // テーマカラー
  format: "book" | "card";       // 表現形態
  patterns: Pattern[];           // 収録パターン一覧
  createdAt: Date;
}

// 個別のパターン
interface Pattern {
  id: string;                    // 例: "hajimari-no-monogatari"
  number: number;                // 例: 1, 10
  name: string;                  // 例: "はじまりの物語"
  intro: string;                 // 導入文・キャッチフレーズ
  illustrationPath: string;      // イラスト画像のパス
  context: string;               // ▼その状況において
  problem?: string;              // しかし〜 / ▼問題
  solution: string;              // ▼そこで
  consequence: string;           // ▼その結果
  note?: string;                 // エピソード・ノート
  tags?: string[];               // 分類タグ
  relatedPatternIds?: string[];  // 関連パターン
  languageId: string;            // 所属するランゲージのID
}
```

## 6. Webアプリで重要な設計ポイント

### A. イラストの優先度

パターン・ランゲージにおいて、**イラストは単なる装飾ではありません**。
テキストを読む前に概念を直感的に理解させる「視覚的アンカー」です。

- カード表示時：イラストを最も目立たせる
- 詳細表示時：テキストより先にイラストが目に入る配置

### B. 2つのビュー

1. **Language View（ランゲージ単位）**
   - 1つのパターン・ランゲージを「本」のように読む
   - 番号順、テーマカラー適用

2. **Atom View（パターン単位）**
   - 異なるランゲージのパターンをフラットに混ぜる
   - 偶発的な発見を促す

### C. 出典の控えめな表示

パターンの「主役」は、パターン名とイラストです。
どのランゲージに所属するかは、アイコンや控えめなラベルで示し、読者の視線を邪魔しないようにします。

## 7. パターン例

### 例1: 楽天主義「はじまりの物語」

| 要素 | 内容 |
|------|------|
| 番号 | 01 |
| 名前 | はじまりの物語 |
| 導入文 | 背景と目的を知り、物語の一員になる。 |
| 状況 | 新しくチームに参加した。 |
| 問題 | 担当範囲の仕事に集中しているだけでは、「チームとして仕事をする」ことにはならない。 |
| 解決 | 背景と目的を聞くことで、現在に至ったのかを理解し、その流れの一部になる。 |
| 結果 | チームで目指していることが共有されていると、同じ方向を向いて仕事を進めやすくなる。 |

### 例2: 燕市「大きな声を出せる場」

| 要素 | 内容 |
|------|------|
| 番号 | 10 |
| 名前 | 大きな声を出せる場 |
| 導入文 | 皆が活発に企画を考えられる場にしたいと思っています。 |
| 状況 | 会議室のように整然とした部屋だと、メンバーたちは緊張したり遠慮したりしてしまい、話しにくくなってしまいます。 |
| 解決 | 机を少しラフに配置し、お菓子やジュースを並べ、皆がくつろぎやすいようにすることで、思いっきり話したり笑ったりできるようにします。 |
